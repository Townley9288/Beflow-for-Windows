# Beflow for Windows v1.0.5.0

本次版本加入完整的原生影视重命名工作流，并集中改进页面布局、设置反馈和下载目录联动。

## 新功能

- 侧边栏新增“重命名”“命名模板”页面，使用 C#、WinUI 3 和 ffprobe 原生实现，不依赖旧 FilmNamer、Python 或 MediaInfo。
- 普通下载会按视频标题组织输出目录，完成后可直接打开文件夹或带入本次视频进入重命名页。
- 支持 TMDB 电影与剧集搜索，获取中英文片名、年份、季与集名；API Key 和代理只保存在本地数据目录。
- 支持分辨率、DV/HDR、编码、帧率、音频格式和声道识别，并可联动重命名字幕、弹幕、封面和 NFO。
- 新增独立命名模板管理，支持字段编排、分隔符、实时示例、自定义模板、内置模板保护及剧集/电影分别记忆。
- 新增重命名历史和持久撤销；执行采用同目录两阶段改名，发生冲突或失败时回滚且不会覆盖已有文件。

## 修复与改进

- 设置页增加 TMDB 配置，并将保存、验证和清理结果改为窗口顶部居中的浮动通知。
- 修复默认下载目录以及 aria2c、mkvmerge 路径通过选择器修改后界面不刷新的问题。
- 设置页和关于页会随窗口宽度继续扩展；重命名与模板页面的字段和卡片布局支持动态重排。
- 命名模板页会正确同步侧边栏选中状态；进入模板或历史页再返回时保留当前重命名工作区。
- 重命名页高级设置恢复为基本信息标题栏右侧的紧凑入口，并改进季数、起始集数和焦点交互。
- 工具检测新增 ffprobe，正式构建会校验三个重命名页面的 WinUI 资源是否完整。
- 本地配置、重命名历史、TMDB Key、任务日志和 B 站登录数据均不会写入 Git 或发行包。

安装版覆盖升级和便携版更新都会保留现有配置、主题选择、下载/重命名历史、任务日志、TMDB 设置及 B 站登录数据。

发行文件暂未进行商业代码签名，Windows SmartScreen 可能显示未知发布者。请从本 Release 下载，并使用对应 `.sha256` 文件核验完整性。
